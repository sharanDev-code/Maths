RAJ MATHEMATICS INSTITUTE - CONTACT FORM IMPLEMENTATION GUIDE
========================================================================

I have implemented a secure contact form with email verification using 
Next.js, Nodemailer, and Framer Motion. 

------------------------------------------------------------------------
1. ARCHITECTURE OVERVIEW
------------------------------------------------------------------------

A. Backend Utilities (lib folder):
   - mailer.ts: Configures Nodemailer with SMTP settings from environment 
     variables. It provides a reusable 'sendEmail' function.
   - verification-store.ts: A server-side in-memory storage that maps 
     emails to their respective 6-digit OTP codes and expiration times. 
     (Note: In a production scale-up, this would typically move to Redis).

B. API Routes (app/api folder):
   - /api/contact/verify/send: 
     1. Generates a random 6-digit code.
     2. Stores it in the Verification Store.
     3. Emails the code to the user using a beautiful HTML template.
   - /api/contact/verify/confirm:
     1. Matches the user-provided code with the one in the internal store.
     2. Marks the email as 'verified' in the store for 10 minutes.
   - /api/contact/send:
     1. Checks if the incoming request's email is already verified.
     2. If verified, sends the final contact message to the Institute's email.
     3. Clears the verification data to ensure security.

C. Frontend UI (app/contact/page.tsx):
   - State Machine: The form uses a 'status' state (idle -> sending_otp -> 
     otp_sent -> verified -> sending_message -> success).
   - Dynamic UI: The OTP input field animates into view only after a code 
     is sent. The "Send Message" button is locked until verification is 
     successful.

------------------------------------------------------------------------
2. ENVIRONMENT VARIABLES (.env.local) EXPLAINED
------------------------------------------------------------------------

Open your .env.local file. You must fill these variables for the system to work:

EMAIL_SERVER_HOST: 
   The SMTP address of your email provider. 
   - For Gmail: smtp.gmail.com
   - For Outlook: smtp.office365.com

EMAIL_SERVER_PORT: 
   The port for secure email sending. 
   - Usually 587 (TLS) or 465 (SSL).

EMAIL_SERVER_USER: 
   The full email address that will be sending the emails (e.g., your-id@gmail.com).

EMAIL_SERVER_PASSWORD: 
   IMPORTANT: If using Gmail, do NOT use your regular password. 
   Instead, go to Google Account Settings -> Security -> 2-Step Verification 
   -> App Passwords. Generate a "Mail" app password and paste it here.

EMAIL_FROM: 
   The 'From' name/address that appears in the inbox. Usually the same 
   as EMAIL_SERVER_USER. This is also where the institute will receive 
   contact submissions.

VERIFICATION_SECRET: 
   A random long string used for internal security signatures.
